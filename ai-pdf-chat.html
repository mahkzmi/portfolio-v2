Project Overview

Traditional document search is rigid — you can't ask questions or explore context. AI PDF Chat bridges the gap between unstructured PDFs and natural language understanding using Retrieval-Augmented Generation (RAG). It lets users query PDFs conversationally with semantic search, retrieving the most relevant chunks and generating precise, context‑aware answers.

Project Challenges

Context retrieval

Challenge: Retrieving the right chunk size while preserving semantic coherence.
Solution: Optimized chunk size with overlap and re‑ranking using cross‑encoders.

Embedding quality

Challenge: General‑purpose embeddings sometimes miss domain‑specific nuance.
Solution: Fine‑tuned Sentence‑BERT on a custom dataset of technical PDFs.

Chunk size

Challenge: Too small = loss of context; too large = noise in retrieval.
Solution: Dynamic chunking based on sentence boundaries and semantic similarity.

Deployment

Challenge: Balancing latency and cost for vector search and LLM inference.
Solution: FAISS index quantization and caching frequent queries.

What I Learned

Retrieval-Augmented Generation

RAG is not just about connecting a vector DB to an LLM. It's about careful orchestration, prompt design, and evaluation to reduce hallucinations.

Prompt Engineering

Crafting prompts that include retrieved context, instruct the model to cite sources, and handle "not found" scenarios gracefully is an art.

Backend Architecture

Building a robust async API with FastAPI, managing state, and streaming responses requires careful design for real‑world usage.
